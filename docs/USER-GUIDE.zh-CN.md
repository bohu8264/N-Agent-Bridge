# N Agent Bridge 安装与使用

本说明适用于 `0.9.9 (24)`。日常界面为“概览、按键、灯光、设置”四页。当前完整验证型号为 NuPhy Air75 V3，后续型号会以独立设备 Profile 加入。

## 1. 安装

1. 双击 `NAgentBridge.dmg`。
2. 将 `N Agent Bridge.app` 拖入 `Applications`。
3. 正式 `NAgentBridge.dmg` 已使用 Developer ID 签名并通过 Apple 公证，正常双击应用即可；第一次只需确认 macOS 的标准“从互联网下载”提示。如果文件名包含 `Development`、系统提示身份未验证，或应用意外退出，就不是可分发正式包，请不要继续安装。
4. 关闭主窗口不会退出桥接；需要停止时，从菜单栏状态图标选择“退出”。

当前版本安装到 `/Applications/N Agent Bridge.app`。旧版 Air75 Agent Bridge 会在升级时保留为带版本号的备份，用于必要时恢复。

## 2. 四个页面分别做什么

- **概览**：查看 Air75 连接、Codex 控制、系统权限和顶部任务侧灯状态；需要处理问题时会直接显示操作按钮。
- **按键**：查看或更改 12 个 Codex 动作对应的实体键，并查看旋钮的三个推理控制动作。
- **灯光**：选择普通背光和侧灯灯效，设置键盘多久无操作后熄灯，开启 Codex 五状态侧灯，并分别自定义五种状态颜色。
- **设置**：管理输入监控、辅助功能、Codex 快捷键、登录启动与恢复操作。

旧版中的设备明细、旋钮、语音、权限、诊断等独立页面已经合并或隐藏。日常使用不需要理解 HID Usage、VID/PID、Profile 字节或备份文件结构。

## 3. 第一次启用

1. 用可传输数据的 USB-C 线连接 NuPhy Air75 V3。
2. 打开应用。首次快速设置会依次显示“连接 Air75 V3”“允许系统权限”“启用 Codex 控制”。
3. 允许两项必需权限：
   - **输入监控**：让 Bridge 只读取 Air75 V3 的专用 F 键和旋钮信号；
   - **辅助功能**：让 Bridge 把对应操作发送给当前 Codex Desktop。
4. 首次在系统设置中打开权限后，完整退出并重新打开 N Agent Bridge，让 macOS 使授权生效。0.9.0 起使用固定 Bundle ID 与固定证书；以后由本项目构建的新包保持同一权限身份，不应再因版本更新而要求反复授权。
5. 点击“启用”。应用会先备份键盘完整键位表，再写入 Air75 专用控制层并安装 Codex 中继快捷键。
6. 写入完成后，将 Air75 关机再开机，或拔掉 USB 后重新连接一次，让键盘固件重新载入板载配置。
7. 用 `Command-Q` 完整退出 Codex Desktop，再重新打开 Codex。这个重启只需在首次安装或修复快捷键后执行一次。
8. 保持 N Agent Bridge 在后台运行。此后直接在当前 Codex 窗口使用键盘，无需在 Bridge 中另建 Agent 或会话。

macOS 不允许安装包替用户静默开启输入监控和辅助功能，第一次必须由当前 Mac 用户亲自确认。F11 使用 Codex 自带听写，麦克风权限由 Codex Desktop 自己申请。

## 4. F1–F12 在 Codex 中的功能

键帽仍然是 F1–F12。应用在键盘内部把它们转换成不会与普通媒体键冲突的唯一事件，由 Bridge 捕获后发送给 Codex Desktop：

| 实体键 | Codex Desktop 动作 |
|---|---|
| F1–F6 | 切换到 Codex 任务 1–6 |
| F7 | 切换 Fast Mode |
| F8 | 批准当前请求 |
| F9 | 拒绝当前请求 |
| F10 | 新建任务 |
| F11 | 启动 Codex 自带语音听写 |
| F12 | 发送 Composer 中的内容 |

F1–F6 对应 Codex 左侧当前任务列表的前六项，并不是 Bridge 内部的六个 Agent。F8/F9 只在 Codex 正在显示批准请求时生效；F12 在输入框有内容时发送。

F11 不依赖 Codex 接收普通 F11。实体 F11 的专用信号会按原始按下/松开时长定向转发为 Codex 提示的 `Control + Shift + D`：短按可启动听写，长按也会保留完整时长。

### 把动作改到数字键、字母键或其他键

1. 打开“按键”页，在要修改的动作卡片上点“更改实体键”。
2. 按一下并松开新的实体键，例如数字 `1`、`2`、`3`、字母键、F 区或方向键。
3. 如果该键已分配给另一个动作，应用会交换两个动作的实体键，保证一个键不会触发两个 Codex 动作。
4. Codex 控制开启时，新键是专用控制键，原字符不会同时输入；停止控制或退出 Bridge 后恢复正常输入。媒体键不开放学习，因为 macOS 无法可靠隔离其系统音量/播放功能。
5. 点“恢复 F1–F12”可随时回到默认实体 F 区。

## 5. 旋钮控制推理深度

旋钮已经合并到“按键”页：

| 旋钮动作 | Codex Desktop 动作 |
|---|---|
| 向左旋转一格 | 降低一级推理深度 |
| 按下旋钮 | 打开模型与推理深度选择器 |
| 向右旋转一格 | 提高一级推理深度 |

Bridge 会先消费键盘的专用旋钮信号，再转换成 Codex 命令，因此启用状态下不应同时改变屏幕亮度或系统音量。推理深度已经到最低或最高时，继续旋转不会再改变。

## 6. 如果按键没有反应

按下面顺序检查：

1. 打开“概览”，确认 Air75 V3 已连接、Codex 控制已配置、系统权限已允许。
2. 在“设置”页确认“输入监控”和“辅助功能”均为“已允许”。若不是，打开对应系统设置，开启 `N Agent Bridge` 后完整退出并重新打开应用。
3. 在“设置”页确认“控制快捷键”为“已就绪”；如果显示“需要修复”，点击“修复”。
4. 打开 Codex 的输入区域并按实体 F11；如果 Codex 开始听写，软件中继已经生效。
5. 如果应用提示需要重启 Codex，请用 `Command-Q` 完整退出 Codex Desktop，而不是只关闭窗口，然后重新打开。
6. 如果概览显示“控制已暂停”，点击“启用控制”。首次写入或重新写入板载控制层时必须连接 USB-C。
7. 如果顶排或旋钮仍同时改变屏幕亮度，请重新检查两项系统权限并重启 Bridge；如果旋钮仍只调音量，请用 USB-C 连接键盘后重新启用控制，再让键盘断电重连。
8. 如果旧版“按键”页曾显示 `P07 / UFFFFFFFF`，它不是实体键。0.9.5 会在启动时自动恢复该动作的默认实体键；无需重新授权，也不要把它作为自定义键保存。

应用会保留 Codex 默认的 `Command-1…6`、`Command-N`、`Enter` 等快捷键，也会保留用户已有且不冲突的自定义设置。修改前的 Codex 快捷键和键盘完整键位表仍会自动备份，但这些工程文件不再显示在日常界面中。

## 7. 蓝牙日常使用

先用 USB-C 完成一次板载控制层写入与回读，再拔线切换到 Bluetooth 或 2.4G。第一次发现蓝牙设备时，应用只会询问是否连接这台 Air75 V3。确认后，板载 F1–F12 和旋钮事件会继续使用已经写入的配置。

2.4G 模式下请把随附的 U1 接收器插入 Mac，并按任意键唤醒；应用只根据该活动来自哪个 HID 接口确认通道，不读取或保存普通按键内容，然后切换到“2.4G 接收器”并补发当前任务颜色。USB-C 只用于充电且仍被系统枚举时也不再抢占 2.4G 路由。

“灯光保持时间”属于键盘管理设置，修改时仍需临时连接 USB-C；这不会影响已经通过 2.4G 实时同步的 Codex 状态侧灯。

蓝牙模式下 F1–F12 和旋钮可以继续使用，但当前 Air75 V3 固件没有蓝牙灯光配置通道，因此无法实时接收侧灯 RGB。需要无线状态侧灯时请使用 2.4G；这不是 macOS 权限问题。

## 8. 键盘灯光

1. 用 USB-C 数据线连接键盘，进入“灯光”页。
2. “普通背光”可选择灯效、亮度和常亮颜色。拖动亮度后停下约四分之一秒，应用会写入两个板载 Profile 并回读确认；不需要额外点击“应用”。
3. 在“灯光保持时间”选择 3、6、10、20、30 或 60 分钟；键盘达到该无操作时间后会熄灯并进入自动休眠。选择“始终亮着”会关闭自动休眠，在蓝牙或 2.4G 模式下会明显增加耗电。
4. 打开“Codex 状态侧灯”后，侧灯显示最近六个任务中最需要关注的状态，优先级为报错、待确认、推理、完成、空闲：
   - 白色：空闲；
   - 蓝色：正在思考或推理；
   - 绿色：任务完成；
   - 橙色：需要你确认；
   - 红色：任务报错。
5. 绿色保留 60 秒、红色保留 120 秒后回到白色；橙色会保持到确认完成，任务继续后自动转蓝。
6. 在“状态颜色”中点击任意色块，可分别更改五种颜色；点“恢复默认”可回到白、蓝、绿、橙、红。颜色会保存在本机。
7. 状态模式只写侧灯，普通背光灯效、亮度、速度和颜色不会改变。关闭状态模式后，可继续选择普通侧灯灯效和亮度。
8. 蓝牙模式不支持实时 Vendor HID 灯光写入。未插 USB-C 时可以先保存颜色，下次连接后自动应用；实时状态变化仍需要 USB-C 或已验证的 2.4G 配置通道。

USB 整键背光、侧灯、亮度和灯效已经实机验证。单键 RGB 与蓝牙实时灯光协议未公开，因此本产品不会把它们显示为已支持功能。

## 9. 恢复与分发说明

“设置”页的“高级操作”可以重新显示快速设置、恢复首次灯光，或在 USB-C 已连接时恢复键盘原始设置。首页、设置页和菜单栏中的“停止并恢复键盘”都会执行同一安全流程：保持数据线连接，等待“现在可以拔掉数据线”的提示，再拔线。应用会拒绝把已经包含 F13–F24 专用层的历史备份当成原始键位。

项目中的 `NAgentBridge-<版本>-Development.dmg` 使用固定本机签名，只适合开发验收和已知来源测试。对外正式发行只能发送完整通过 Developer ID、Apple 公证和 Stapling 的 `NAgentBridge.dmg`；当前正式包仍等待 Apple Developer 证书和公证 profile。
