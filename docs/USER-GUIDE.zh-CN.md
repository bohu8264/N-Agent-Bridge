# N Agent Bridge 0.14.0 使用说明

当前版本只支持 **NuPhy Air75 V3 ANSI**，推荐使用官方固件 **1.0.16.6**。

## 安装

1. 下载 `NAgentBridge-0.14.0-Development.dmg`。
2. 把 `N Agent Bridge.app` 拖入“应用程序”，再弹出 DMG。
3. 第一次打开若被 macOS 拦截，到“系统设置 → 隐私与安全性”点击“仍要打开”。不要运行 `spctl --master-disable`。
4. 在“隐私与安全性”分别允许 N Agent Bridge 的“输入监控”和“辅助功能”。
5. 完整退出并重新打开 N Agent Bridge。

权限开关属于 macOS 安全机制，应用无法替用户点击，也不能合并成一个授权。

## 首次配置

1. 在 NuPhyIO 更新 Air75 V3 后，关闭配置页面或浏览器标签页。
2. 把键盘切到有线模式，用能传输数据的 USB-C 线连接。
3. 打开 N Agent Bridge，确认左下角显示“NuPhy Air75 V3 已连接”。
4. 点击“连接并启用”。
5. 等待应用显示 F13–F24 已回读确认；处理中不要拔线或重复点击。

应用会先读取并备份完整 1568-byte 键位表，再把物理 F1–F12 转为专用 F13–F24，最后逐字节回读。这样按键只触发 Codex，不再同时触发屏幕亮度、音量或媒体功能。固件升级恢复原生 F1–F12 后，重新用 USB-C 点一次“连接并启用”即可修复。

## 默认控制

- F1–F6：六个 Agent 任务
- F7：Fast Mode
- F8：批准
- F9：拒绝
- F10：新建任务
- F11：Codex 原生听写
- F12：发送
- 旋钮左转 / 按下 / 右转：降低推理深度 / 打开选择器 / 提高推理深度

在“按键”页可把 12 个动作改到数字、字母、F 区或导航键。Agent 1–6 改键后，对应的单键状态灯会跟随新的实体位置。

## Agent 与灯光

Agent 来源有四种：最近对话、置顶对话、优先对话、自定义分配。应用按 Codex 对话 ID 绑定，因此侧栏重新排序不会让按键和灯光错位。

状态颜色：

- 白：空闲
- 蓝：思考/推理
- 绿：完成
- 橙：等待用户确认
- 红：报错

“灯光”页可以选择普通背光、侧灯灯效、常亮颜色和灯光保持时间。Agent 状态只写对应实体键，不接管侧灯。

## 连接方式

- USB-C：全部功能；首次配置和恢复必须使用。
- 官方 U1 2.4G：按键和状态灯可用，前提是键盘/接收器固件转发 S4 配置命令。
- 蓝牙：按键可用；当前官方固件没有实时单键灯光配置通道。

## 停止使用或恢复键盘

使用 USB-C 连接后，在应用中选择“停止并恢复键盘”。应用会从有效备份写回原始键位并完整回读，确认成功后才可以拔线。不要删除 `~/Library/Application Support/Air75AgentBridge/Backups`。

## 常见问题

### 一直显示“USB-C 待响应”

确认键盘为 Air75 V3 ANSI、固件为 1.0.16.6、有线模式和数据线正常，并完全退出 NuPhyIO。点击右上角刷新；仍失败时重新插拔键盘后重启应用。

### 权限已开但应用仍提示未允许

确认授权列表中是 `/Applications/N Agent Bridge.app`，不是旧副本或直接从 DMG 打开的 App。关闭再打开权限开关，然后完整退出 App 重试。每次更新请覆盖同一个“应用程序”内的 App，不要保留多个副本。

### F 区仍同时执行 macOS 功能

这说明键盘固件中的专用层尚未写入或固件升级后被恢复。使用 USB-C 再点一次“连接并启用”，直到应用确认 F13–F24 已回读。

### 安装包无法直接打开

免费 Development 包未经过 Apple 公证。使用“隐私与安全性 → 仍要打开”；不要关闭整个 Gatekeeper。Apple 无警告的站外发行需要付费 Apple Developer Program、Developer ID 和公证。
